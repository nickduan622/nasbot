# M-Team (馒头) 新手实战指南

---

## 一、新手考核（前 28 天，生死期）

注册后有 **28 天**考核期，达成以下**任一条件**即可通过：

| 条件 | 目标 |
|------|------|
| 上传量 | > 20 GB |
| 下载量 | > 15 GB |
| 魔力值 | > 4,500 |

### 红线（触发即封号）
- 下载量超过 10 GB 时，分享率低于 **0.3** → **立刻封号**
- 注册后 **5 天零活动** → 自动删号
- **40 天**不登录网站（光做种不算）→ 删号

## 二、新号第一周行动计划

### Day 1（今天就做）
1. **登录 M-Team 网站**，确保活动记录不为零
2. 在网站浏览页面筛选 **Free** 或 **2xFree** 标签的种子
3. 下载 5-10 个 Free 种子（大小不限），开始做种
4. **绝对不要碰非 Free 的种子** — 你现在没有上传缓冲，下载正常计费的种子会让分享率暴跌

### Day 2-7
5. 继续抓 Free 资源，目标：累计做种 **500GB+** 的内容
6. 保持 qBittorrent 24/7 运行
7. 每周至少登录一次 M-Team 网站（光做种不算登录）

### ⚠️ 考核期禁忌
- **不要用魔力值兑换上传量** — 考核看的是累计获得的魔力值，花掉就不够了
- **不要下载非 Free 种子** — 直到你有足够的上传缓冲（比如上传了 50GB+）
- **不要用 qBittorrent Enhanced Edition** — 在黑名单上，会被封号
- **不要开 VPN 连接 tracker**
- **不要开匿名模式** — tracker 无法识别你，上传不计分

## 三、分享率维护策略

### 优先级：下载什么种子
```
2xFree（最优）> Free > 2x 50% > 50% > 2x > 30% > 普通（新手禁碰）
```

- **2xFree** = 下载不计量 + 上传双倍 = 无脑抓
- **Free** = 下载不计量 = 安全
- **普通** = 下载计量 = 新手慎重

### 做种要求（每个种子）
- 做种 **48 小时**，或达到 **1:1 分享率**（先到的算）
- 必须在 **7 天内**完成
- **Hit & Run（下完就删）= 严重违规**

### 魔力值怎么攒
每小时按做种情况自动结算，影响因素：
- 文件越大 → 越多
- 做种人越少 → 越多
- 种子越老 → 越多
- 做种数量超过 100 个 → 额外 +5% 每 100 个（无上限）
- 开了二步验证 → +10%

**实操建议**：多挂小文件（电子书、小软件），凑到 100+ 个做种数，触发阶梯奖励。

## 四、qBittorrent PT 专用设置

### Connection（连接）
| 设置 | 值 | 原因 |
|------|-----|------|
| Listening Port | 选一个 45000-65535 的固定端口 | 需要在路由器做端口转发 |
| UPnP / NAT-PMP | **关闭** | 手动转发更可靠 |
| Global connections | 200-500 | |
| Connections per torrent | 50-100 | |

### BitTorrent 协议（关键！）
| 设置 | 值 | 原因 |
|------|-----|------|
| **DHT** | **关闭** | PT 站禁止，开了可能被标记 |
| **PEX** | **关闭** | 同上 |
| **Local Peer Discovery** | **关闭** | PT 不需要 |
| **Anonymous Mode** | **关闭** | 开了 tracker 不认你，上传白做 |
| Encryption | Allow encryption | |

### Speed（速度）
| 设置 | 值 | 说明 |
|------|-----|------|
| Upload limit | 不限 或 80% 带宽 | 不要限制上传，影响分享率 |
| Download limit | 不限 | |

### Seeding（做种）
| 设置 | 值 | 说明 |
|------|-----|------|
| When ratio reaches | **2.0** | 安全值 |
| When seeding time reaches | **4320 分钟**（72小时） | 超过 48h 要求的缓冲 |
| Then | **Pause torrent**（暂停，不删除） | 保留文件用于 hardlink |
| Max active uploads | **不限或很大** | 尽可能多做种 |
| Max active downloads | 3-5 | |
| Do not count slow torrents | **开启** | 慢种不占活跃配额 |

### 端口转发（回家后在路由器设置）
1. 登录路由器后台
2. 找到「端口转发」或「虚拟服务器」
3. 把 qBittorrent 的监听端口（比如 55000）从外网转发到 NAS 的内网 IP:端口
4. 端口可连通后，qBittorrent 状态栏会显示绿色图标
5. **有端口转发 = 能被其他 peer 主动连接 = 上传量显著增加**

## 五、*arr 自动化与 PT 的协作设置

### Radarr/Sonarr 中的关键设置

**Download Client（下载客户端）配置：**
- **不要勾选** "Remove Completed Downloads" — PT 需要持续做种
- Category 设好（radarr / sonarr）

**Indexer（索引器）配置（高级选项）：**
| 设置 | 值 |
|------|-----|
| Seed Ratio | 2.0 |
| Seed Time | 2880 分钟（48小时） |

这告诉 Radarr/Sonarr：这个 PT 站的种子至少做种到 2.0 比率或 48 小时后才算"完成"。

**Media Management：**
- **Use Hardlinks instead of Copy** = 开启（最重要的一个设置）
- 这样下载的原文件留在 downloads/ 继续做种，media/ 里的是硬链接，不占额外空间

### Prowlarr 中添加 M-Team
- Indexer 类型：搜索 "M-Team"
- API 地址：`https://api.m-team.cc`（注意是 api 子域名）
- 需要填入你的 **Passkey**（在 M-Team 网站个人设置里找）

### 建议的自动化策略

**考核期（前 28 天）：不要用 Radarr/Sonarr 自动下载**
- 手动在 M-Team 上挑 Free 种子下载
- 先积累上传缓冲
- Radarr/Sonarr 可以先配好，但不要启用自动搜索

**考核通过后：**
- 开启 Radarr/Sonarr 自动搜索
- Prowlarr 优先搜索 Free 资源（如支持 freeleech 过滤）
- 让系统全自动运行
